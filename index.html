Overview

This course covers principles and practices of software engineering alongside project management techniques used to plan, execute, and deliver software systems. Topics include requirements, design, testing, agile methods, risk management, estimation, and team collaboration.

Syllabus (high level)

  • Introduction to Software Engineering & lifecycle models
  • Requirements engineering and stakeholder analysis
  • Testing, quality assurance, and continuous integration
  • Project planning, scheduling, and estimation (e.g., COCOMO, story points)
  • Agile frameworks (Scrum, Kanban) and team practices
  • Risk, change, and configuration management

Learning Outcomes

  • Explain major software development lifecycle models and trade-offs
  • Create basic requirements artifacts and use cases
  • Design simple software architectures and apply design patterns
  • Plan and track a software project using common project management techniques
  • Apply basic testing strategies and quality assurance practices

Resources

  • Suggested text: Ian Sommerville, Software Engineering
  • Online: Scrum Guide, ISO/IEC standards summaries
  • Tools: Git, Jira/ClickUp, CI services (GitHub Actions)
  • Supplemental: Software architecture and design patterns references
